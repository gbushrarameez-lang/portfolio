Case study — MidCost (Hospital Price Transparency)

Hesten Solutions · 2025 — Present

Overview: MidCost ingests nationwide hospital price-transparency MRF files, normalizes inconsistent pricing rows and serves high-performance, comparable procedure-level price search across hospitals, payers and plans.

Problem

  • Files differ widely in schema and structure; large (GB+ per hospital).
  • Ambiguous payer/plan/procedure mappings; relational-only solutions are cost-inefficient at national scale.

High-level architecture & data flow

Layer 0 — Lossless Staging
Explode raw files, preserve payload, file hash, URL and row-level errors.
Layer 1 — Canonical Atomic Prices
Normalize to atomic model: hospital × service × payer × plan × price-type.
Layer 2 — Aggregated & Serving
Aggregations (min/median/max); Parquet outputs and search-optimized serving (embeddings + filters).
Storage
Parquet on S3-compatible object storage; MySQL for canonical transactional indexes.

Impact snapshot

  • 5,400+ hospitals ingested (MidCost production)
  • Multi-TB canonical datasets (Parquet)
  • Sub-second search for common queries
  • ≈90% archival storage cost reduction using Parquet + object storage

Technical components

  • Python ETL (streaming & batch), SQLAlchemy, custom CSV/JSON/ZIP parsers.
  • Deterministic canonicalization engine; payer-plan resolution and deduplication.
  • Parquet writer + partitioning strategy; S3-compatible object store for scale.
  • Embedding-based search for service description matching and ranking.

Projects

MidCost — Hospital Price Transparency & Search Platform

Company: Hesten Solutions • Duration: 2025 – Present

Nationwide healthcare data platform ingesting and normalizing U.S. hospital price transparency MRF files at national scale.

MidCost ingests gigabyte-scale machine-readable files (MRFs) published by U.S. hospitals, normalizes highly inconsistent schemas, and preserves full provenance for auditability.

A multi-layer canonical ETL pipeline stores raw payloads, file hashes, validation errors, and metadata before resolving data into an atomic pricing model (hospital × service × payer × plan × price-type).

Canonical data is exported to Parquet on object storage and served through a search-optimized layer enabling sub-second price discovery nationwide.
